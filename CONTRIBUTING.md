## Contributing to PyTorch OSPO and Academic Working Group Resources

We want to make contributing to this project as easy and transparent as possible!

## How to make pull requests

- Fork the repository and create your branch from `main`.
- Ensure your contribution is aligned with the goals and scope described in the charter.
- Open a pull request with a clear title and description of what changed and why.

## How to contribute through issues

- Use issues to suggest improvements, propose new work, report problems, or start discussions.
- Please provide enough context for others to understand the need and the proposed direction.
- If applicable, link related pull requests.

## Types of contributions

Contributions may include:

- documentation improvements,
- curriculum and workshop materials,
- templates and checklists,
- implementation ideas,
- feedback on working group outputs,
- fixes to repository content or structure.
